# Учебный проект    

Проект интернет-магазина, реализующий базовые сущности «Товар» и «Категория» с автоматическим подсчётом общего числа категорий и товаров. Данные загружаются из JSON-файла. Код покрыт модульными тестами.

## Описание
Проект демонстрирует:
- Создание классов `Product` и `Category` с заданными атрибутами.
- Автоматическое ведение счётчиков количества категорий и товаров (атрибуты класса).
- Чтение данных из JSON-файла

## Реализованный функционал

- **Класс `Product`** — хранение информации о товаре: название, описание, цена, количество в наличии.
- **Класс `Category`** — хранение информации о категории: название, описание, список товаров (объектов `Product`).
- **Атрибуты класса `Category`**:
  - `total_categories` — общее количество созданных категорий.
  - `total_products` — общее количество товаров во всех категориях (длина списков продуктов при создании).
  Счётчики увеличиваются автоматически при создании нового экземпляра `Category`.
- **Загрузка данных из JSON-файла** — функция `load_categories_from_json` в `src/utils.py` читает файл формата JSON и преобразует его в список объектов `Category` с вложенными `Product`. Путь к файлу передаётся аргументом (по умолчанию `data/products.json`).
- **Демонстрационная точка входа** — `main.py` загружает данные из JSON и выводит сводку по категориям и товарам в консоль.
- **Набор тестов** (pytest):
  - корректная инициализация `Product`;
  - корректная инициализация `Category` (с товарами, с пустым списком, с `None`);
  - подсчёт количества категорий и товаров;
  - проверка загрузки из реального JSON-файла (`data/products.json`).
  Все тесты изолированы друг от друга благодаря фикстуре, сбрасывающей счётчики перед каждым тестом.
- **Линтер flake8** с конфигурацией, исключающей виртуальное окружение и задающей максимальную длину строки 120 символов.
